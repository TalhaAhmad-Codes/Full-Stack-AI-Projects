# Project 2 — File Analyzer Tool

---

# Problem Statement

Build a console tool that analyzes text files and generates reports.

The tool should:

* read files
* count words
* count lines
* detect duplicate words
* calculate statistics
* search keywords
* generate summaries

---

# Concepts Covered

Main focus:

* File Handling
* Async/Await
* Exception Handling
* LINQ
* Dictionaries
* Extension Methods
* Performance basics
* Span/ReadOnlySpan (optional)

---

# Suggested Features

## Core Features

* Read text file
* Count:

  * words
  * characters
  * lines
* Find most common words
* Search keyword frequency

## Intermediate Features

* Ignore stop words
* Case-insensitive analysis
* Export analysis report
* Analyze multiple files

## Optional Features

* Real-time monitoring
* Directory scanning
* CSV export

---

# Suggested Folder Structure

```text
FileAnalyzer/
│
├── Models/
├── Services/
├── Utilities/
├── Reports/
└── Program.cs
```

---

```text
AnalyzeFileAsync()

```

---

## ReportGenerator

Methods:

```text
GenerateSummary()

```

---

## TextProcessor

Methods:

```text
CleanText()

```

Good place for:

* extension methods

---

# Data Structures to Practice

Use:

* Dictionary<string, int>
* HashSet<string>
* List<string>

---

# Functional Flow

```text
User Selects File
    ↓
Read File Async
    ↓
Process Text
    ↓
Generate Statistics
    ↓
Display Report
    ↓
Export Optional Report
```

---

# Performance Practice

This project is PERFECT for learning:

* memory usage
* allocations
* string optimization

You can later experiment with:

```text
Span<char>
ReadOnlySpan<char>
StringBuilder
```

But keep first version simple.

---

# Important Thinking

Do NOT:

* optimize too early
* use advanced memory APIs everywhere

First:

* make it clean
* make it work
* THEN improve

That is real engineering.

---

# What You Should Learn

After this project:

* async file processing
* text processing
* collection usage
* LINQ in real scenarios
* performance awareness

# Project 3 — Async Log Processing System

---

# Problem Statement

Build a system that simulates real server log processing asynchronously.

The system should:

* generate logs
* process logs asynchronously
* filter errors
* save logs
* analyze statistics
* simulate concurrent processing

This project introduces backend engineering thinking.

---

# Concepts Covered

Main focus:

* Async/Await
* Task
* ValueTask
* Collections
* Exception Handling
* Delegates/Events
* File Handling
* Memory basics
* Performance thinking

---

# Suggested Features

## Core Features

* Generate logs
* Process logs asynchronously
* Save logs to file
* Filter ERROR logs
* Search logs

## Intermediate Features

* Real-time log monitoring
* Log event notifications
* Parallel processing
* Statistics dashboard

## Optional Features

* Background worker simulation
* Log queue system
* Retry mechanism
